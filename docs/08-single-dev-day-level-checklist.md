# 单人研发 Day 级开发清单书（Phase 1 + Phase 2）

## 适用范围
- 适用对象：1 人团队，兼任开发、研发、测试、运维、文档维护。
- 节奏假设：12 周，5 个工作日/周，共 60 个工作日（Day 1 - Day 60）。
- 执行规则：每天结束前必须完成「代码或配置变更 + 最小验证 + 文档更新」。

## 每日固定动作（每天都做）
- [ ] 更新任务状态（`docs/task_board.generated.csv` 或等效看板）
- [ ] 记录当天执行命令与关键日志摘要
- [ ] 更新风险项（如新增阻断或概率变化）
- [ ] 确认次日唯一优先级任务

## 验收门禁（单人版）
- 每日门禁：当天任务必须有可验证产物（文件、命令输出或文档变更）。
- 每周门禁：必须通过当周出口标准，未通过则下周第一天先补齐。
- 阶段门禁：Phase 1 通过 S1/S3/S4，Phase 2 通过 S1-S5。
