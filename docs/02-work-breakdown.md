# WBS（单人开发/研发）

## 角色定义（单人）
- 角色：1 人兼任开发、研发、测试、运维、文档管理。
- 目标：在不拆角色前提下完成 Phase 1 MVP 与 Phase 2 增强交付。
- 日级执行清单：`docs/08-single-dev-day-level-checklist.md`。

## 模块主线（按执行顺序）
- W1 输入与配置治理：samplesheet/config/schema/校验
- W2 流程编排：Snakemake DAG、模块依赖、profile 调度
- W3 样本级分析：QC、拼接、识别、分型、宿主、注释
- W4 样本间分析：丰度、差异、α/β多样性、网络
- W5 报告系统：HTML/PDF 汇总与 SVG 图产物管理
- W6 稳定性工程：日志、失败重试、断点续跑、回归门禁

## Phase 2 增强任务包
- E1 比较基因组：交换事件识别与结构化输出
- E2 跨样本分箱：聚合策略与可追溯中间结果
- E3 三代增强宿主预测：甲基化分支与降级路径
- E4 统计稳健性：参数敏感性、过滤阈值、重复性验证
- E5 性能优化：资源模板、并发策略、I/O 优化
- E6 图形与报告定稿：网络图、PCoA/NMDS、热图统一风格

## 单人模式时间分配建议
- 每周 60% 开发 + 20% 测试回归 + 20% 文档与发布准备。
- 每周固定半天处理技术债与失败用例。

## 交付门禁
- 每个任务包必须包含：输入、输出、资源预算、失败重试策略、验收标准。
- 每周必须有可复现命令与结果快照。
- 未通过模块级验收，不得进入下一阶段集成。
